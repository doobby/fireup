## What is Computer Programming? 什么是计算机编程
如果你以前从未编写过程序，那么这个话题就像一个巨大、令人不寒而栗的黑洞。当我告诉别人我是一名软件开发人员时，对方通常会一脸茫然，继而转换话题。

但它真的没那么复杂。计算机编程，也称为编程，编码或软件开发（每个术语之间存在细微差别），是编写计算机可执行指令的过程。

比如说：

如果你曾经写过一个Excel宏，你可以说你编写过计算机程序。

如果你曾经构建过一个HTML页面，你可以说你编写过代码（有些人会对此存在一些争论，但是它符合定义）。

如果你曾经使用BASIC，Pascal，C，Java，JavaScript或PHP等主流编程语言编写过代码，那么你毋庸置疑已经进入了计算机编程领域。

### 为什么编程是必要的
我们需要编程是因为计算机并不那么“聪明”。如果没有人告诉它们该做些什么，他们只不过是一大堆硬件和闪光灯。 软件是运行在硬件上的，而软件也正是乐趣所在（对于我，一个软件开发人员来说）。

计算机程序，又叫做应用程序或者计算机软件，是由一个或多个程序员编写的指令集或代码集合，进而在计算机上执行。

代码可以使用任意一种计算机语言（也称为编程语言）来编写，例如C、C++、Java、Visual Basic、Perl、PHP等等。实际上，目前有数百种（数千种？）编程语言，每一种都有自己的特色。但是在任何特定时期，通常只有少数被广泛使用，这些语言往往会流行几年，然后慢慢被新的语言取代。尽管如此，有些语言仍然坚持流行了几十年（比如COBOL，C和C ++）。

软件运行的计算机可以是公司内部的台式计算机、外部客户计算机、Web服务器，也可以是在汽车、电梯和微波炉中运行的其他鲜为人知的专用计算机。

Microsoft Word及其整个办公套件，是运行在我们台式计算机（或笔记本电脑）上的程序。

Amazon.com的网站是一个名为OBIDOS大型定制软件，它运行在世界各地成千上万的亚马逊网络服务器上。当你打开Firefox或Internet Explorer时，它会向其中一个Web服务器发送请求，同时，Obidos的一个实例会回复你请求的信息。

### 不仅仅是编程
虽然编程可以被描述为“编码”的基本行为，甚至更简单的称作“将计算机指令输入编辑器”，但是作为一个程序员不仅仅需要写代码。

软件开发，一个你可能听说过的可以和计算机编程互换的术语，涵盖了从概念到成品的整个过程。虽然编程是输入指令的行为，但是软件开发却包含了创作软件过程中的多种任务，包括：

* 与软件用户（或潜在用户）交流，从而得到新的功能点
* 编写规范文档以描述软件的运行方式
* 与其他程序员讨论功能和设计方法
* 编写代码（即编程） 
* 编写测试
* 测试代码
* 修复错误
* 准备发布软件
* 将其发布到生产环境，并分发给用户
* 等等...

对于许多程序员来说，编程是最令人愉快的部分，但是除了坐在计算机前敲下数以千行的代码之外，你还需要完成上述的诸多步骤。

## Why Should I Become a Programmer?  为什么我要成为一名程序员？

编程（我也把它叫做软件开发，即使它们不完全相同但我会换着用~），是一个富有挑战，高回报的工作。如果你拥有不落伍的技术，工作多到爆。编程工作经常伴随着弹性工作时间，地点（可以在家，咖啡店甚至另一个城市工作），并且你所学的技能可以轻送移植到新岗位上。

### 工作
根据美国劳工部的数据，2000年至2010年，80%的快速增长职位是与计算机相关的。每一本杂志的就业调查中的top10总有几个和计算机搭边。更不用说自90年代以来人们一直在谈论关于合格程序员的短缺的问题。

结果是，有数不清的编程工作！

Dice.com的一份研究表明真的存在着上千个程序员职位缺口（截止目前，有超过20000份工作里用到了“开发者”这个词）

在我还是个赚工资的职员时，我没有当一天闲人。当然，这个也行因人而异，但是如果你住在大城市周边，那必然随时都有大把的工作机会等待着有经验的程序员。

我的一个朋友在洛杉矶是资深.NET程序员，几个月前他把他的简历放到了Dice上，在接下来的三天里他收到了40个电话。热情太高以至于他为了日常工作不得不把简历撤下。

### 硬技能
当市场或销售专业的人寻找工作时常常需要花费数月时间，当一个程序员寻找工作时，通常几天就会收到面试通知。这是部分是由于充足的编程工作，也是因为编程是一项“硬技能”。

硬技能是指像编程，会计，工程类的技能，软技能是指销售，市场，人际交往等技能。两者都很有价值，但硬技能在简历上更容易量化检验，程序员的职位筛查流程会远远快于产品经理。

我保证如果你把拥有资深.NET开发，资深Java开发或者资深php开发（同时拥有相应的实际经验）的简历放到招聘网站上，几个小时内你的电话就会响。

### 高回报
我觉得你绝不能单单为了高工资而选择工作。但如果你喜欢编程（或觉得你会喜欢），同时有人会为你付大钱，这不是挺好吗

花点时间，去Salary.com，输入程序员1和你的邮政编码，在结果里看看，虽然这不是最常见的术语，但他们用“程序员1”去代表有0-3年的程序员，程序员2-5代表更高等级的程序员。

在我的地区（波士顿），入门程序员薪资水平在46k到75k美元之间，大部分在60k左右，对于职场菜鸟来说是份不错的待遇。

在大城市里对于资深程序员来说，拿到125k到150k也不算难。这通常需要5-7年的经验和特定方向相应水平的知识技能。

因为这种人比较难找，高级程序员比菜鸟程序员更容易找到工作

### 弹性制
过去10年中的6年我是在家上班的，剩下的4年我是作为管理者管理团队，或者我在一周的部分时间待在公司。一个有动力的程序员在家会比公司做的更多因为家里收到的干扰更少。更容易测算当前进度，所以经理更倾向于让程序员在家工作，最起码视情况调整。

过去6年中的5年我都和我的老板或者客户住在不同的城市，作为一个程序员让我可以从萨克拉门托搬到洛杉矶到纽黑文再到波士顿，我从没有因为地理原因放弃任何一份工作

你所需要的只是一条高速宽带和一部手机，我写下的数以千计行代码中有在咖啡店留下的，有在山上写的，甚至是在沙滩上

### 挑战性
从圈外人的角度看我上面给出的理由很诱惑，一旦你成为一个程序员，最重要的是干一行爱一行。有时候你会做在电脑面前10小说就为了一个调试一个网页。。。你必须爱它否则他会让你崩溃的甭管你挣多钱。

如果你认识程序员，问他是否喜欢他的工作，回答大致会跟下面的类似：
--很有挑战性，而且我一直能学到新东西。
--调整在于解决问题，有些问题会简单到像“我们如何能给用户一个新按钮让他们可以用来打印”
--其他可能会复杂如“我们有三年的VB.NET的代码想翻译成C#，我们该怎么做？”
但最后，我认识的所有热爱编程的人都爱解决问题，编程真的是每天都有新问题。

### 创造的艺术
原谅我，不过我要开始跟你谈点哲理了。有些人不会在乎这个原因，但这是我热爱编程最主要的原因。

关于编程到底是艺术还是科学一直有很多争论，我个人认为两者皆有，但这都不重要。

最让我上瘾的是每当我码完一堆代码或者第一次尝试做个小样例时在我脑中冲激的脑内啡。第一次从数据库拉回数据并展示出来时，我心中充满了巨大的成就感。我知道这听起来很蠢，但事实上这些我创造的有形的产品（比如网站页面）也许才是过去10年我坚持码代码的最大原因吧

从无到有的创造真有种很棒的感觉。

### 它很酷
好吧，编程通常来说不算酷。多数时候你会做一个只有少数人会用的发票系统，它并不同它听起来一样无聊，但也不会是迪斯尼般有趣。

当然也有为Sundance电影节制作网站（我在2003年做的）的时候，或者在MySpace或者YouTube做高调的工作（像我的朋友那样）。或者你做的网站一年卖了1700万产品（我和我朋友的网站~）。

在那些时候编程可是很有趣的。。。是的，可能成为一个程序员是挺酷的。

## What Are Some Reasons Not to Become a Programmer? 有什么理由阻止你成为一名程序员

### 如果你不喜欢编程
如果你是一个优秀的程序员，你一周起码有40小时的编程时间（而且周末也会思考代码）。如果你不喜欢编程，这工作就不是你的菜。

### 你必须持续提升自己的技能
编程语言和技术每两三年就会改变，如果你没有足够持久的求知欲，编程也不是你的菜。优秀的码农会永远读书，活到老学到老。

### 管理能力
在公司里寻找一个优秀的管理者是困难的，这个问题在IT公司尤为突出。很多IT经理是程序员出身，所以他们比起管理员工本身更偏爱技术，IT公司的经理管出的烂摊子数不胜数。

当然，这也可以成为一些人的的动力，先做开发而后他们可以成为不错的管理者。而优秀的IT经理在商业世界一定是供不应求的。

### 加班 

情况因事而异因时而异，但由于编程都有时间节点，总有你需要加班的时候。对程序员来说加班几乎是没有加班费的。一些工作甚至把这列为日常，大多数工作每隔一两个月就会要求加班。（苦逼）😔

## 编程“世界”都有哪些不同？

不同的人拥有他们不同的软件“世界”。Joel Spolsky，他在知名博客[Joel on Software](https://www.joelonsoftware.com/)中的[定义](https://www.joelonsoftware.com/2002/05/06/five-worlds/)和我略有不同，不过基本观点相似。这里有我如何定义的六类软件世界：

### 产品

包含为大企业例如微软的word或者excel工作，为企业类似谷歌的谷歌地图或者邮件工作，又或者为一个小企业类似Salesforce.com的同名网站应用程序工作。 

产品开发非常具有挑战，经常伴随着很紧的工期和投入之后大量的加班时间。我可以大胆的说很大部分具有天赋的开发者都倾向于产品开发，相比于联合开发可以辩解，这里有最困难的问题和允许你构建更高质量的软件条件。

### 联合开发

典型例如为银行、保险企业、或者其他大企的会计部门、语音中心、船务部等等开发软件。

这种情况下你将使用类似.NET或者JAVA等企业级技术语言，来构建网站应用、桌面应用（典型的例如.NET windows 应用程序）或者手机应用。

在我的职业生涯中曾经当过几年联合开发者。*这里提供了大多数软件开发工作，相比于获取更多经验而言你将获得更高的报酬🆘*。 

### 嵌入式软件

嵌入式软件是将代码烧入一个定制芯片，这类软件运行在你的汽车、电梯控制器和手持GPS设备中。

这个领域我不太熟悉，除了知道这类工作更类似于构建硬件而不是软件，*自从你获得一次成功（一旦产品推出就不需要重新发布）🆘*。这种工作发布和测试周期很长，因为不需要二次开发。我不时地寻找嵌入式软件开发工作职位，在我的经验（十分有限）看来，是一个不错的工作选择。

编写嵌入式软件会很有挑战性，同时你也可以获得很多薪资，但是与联合开发相比它的工作可选性很有限。薪资回报同样趋向于比联合开发低些，

### 游戏开发

*如果你喜爱游戏，构建它会很爽🆘*。它同样有巨大的工作量。我从游戏开发者获取的第一手描述都表明他们热爱这份工作，但是讨厌一周长时间加班（60-80小时）。*相比联合开发，薪酬回报同样趋向较低，这说的通吗？如果你在构建一个收费应用比如光环3你会接受工资削减吗？🆘*

游戏开发需要数学头脑，这是我说需要大学级别数学技能的开发领域之一。

### 咨询

你可以在大型企业类似EDS（电子数据系统公司）、BearingPoint （毕博咨询公司）或者accenture（埃森哲），又或者一个小型咨询事务所（我从事了几年）。咨询拥有许多乐趣，同时趋向于去区分边缘化技术和新的编程技术。但是为大型咨询公司工作需要大量的出差，而小型企业很可能不能为你提供全职工作。

### 自由职业者

一旦你拥有经验也可能开始接触兼职开发项目，最终会引导你成为一名自由开发者。回报富足，但是困难的部分是维持一连串稳定的工作源。

你为自己工作，作为自己的老板拥有完全的自由，但是你不得不为自己买健康保险，当你给自己放一天假你将没有任何回报，同时你必须自己处理市场、财务、退休金等等...

但是对于许多创业开发者，这是极具吸引力和可实现的目标，包括我自己。


## What Programming Language Should I Learn? 我应该学习什么编程语言？

这是一个相当主观的问题，就像问“如果我想成为一名翻译，我应该学习西班牙语，法语还是德语？”如果不了解你对自己成为程序员后的职业规划这个问题就会很难回答。

因为很有可能你在读这本电子书时只会少量的编程语言知识，所以我并不想为你提供学习25种编程语言的可能性。

根据软件开发领域我列出了几个编程语言，通常(注意：我正在进行全面的概括并选择了最流行的语言以减少困惑):
* 桌面程序通常使用 C++，C＃，VB.NET 或 Java 编写
* Web应用通常使用 ASP.NET，PHP，Ruby on Rails 或 Java编写
* 企业开发通常使用桌面或 Web 语言，或两者兼而有之
* 嵌入式软件通常使用 C 或 C++ 的专用版本
* 游戏开发通常使用 C++（最近 C＃ 使用更多）
* 咨询和自由职业者的工作往往以桌面和 Web 语言完成

由于世界上有很多的编程语言，我将假设你的一个优先考虑是在世界上大多数发达地区找到一份工作，*而且你希望投入时间去学习一门可以展示未来几年就业发展的语言* :sos:。

遵循这个逻辑和软件行业中许多人的意见(包括我)，桌面应用程序开发正在消亡。 它尚未死亡，但在大多数情况下，Web 编程是未来发展的方向。 Web 编程是一个增长的行业，它将会存在很长一段时间。

考虑到这一点，这里列出主要的 Web 编程语言：

### <font color=#6495ED size=4 >ASP.NET</font>

ASP.NET是我的首选开发平台（ C＃ 和 VB.NET 是具体的编程语言）。ASP.NET运行在 Windows 或者装有 Windows 的 IIS Web 服务器上。 可以使用 Visual Studio 进行开发，并且通常使用 SQL Server 数据库（两种工具的免费版本都是可用的）。

这是我推荐的路线，因为只要你拥有一台 Windows  电脑就可以轻松的进行程序开发，这门技术可以让你在大型企业，小公司，Web 产品公司找到一份工作或者为自己工作，有很多靠谱的 .Net 工作而且 .Net 也是世界上最流行的编程平台之一。

### <font color=#6495ED size=4 >PHP</font>

PHP是一种开源脚本语言，也是其他语言中唯一你应该考虑的。它非常适合使用免费工具构建网站。 它运行在Apache Web服务器上，通常使用 MySQL 数据库（两者都是开源的）。

PHP 工作机会很多，但没有 .NET 开发人员那么多的高薪工作。另外，企业更有可能使用.NET或Java，而创业公司使用 PHP 或 Ruby on Rails 更多。

### <font color=#6495ED size=4 >Ruby(on Rails)</font>

Ruby on Rails 是一个很酷的新 Web  2.0 框架。Ruby 是一种编程语言，Rails 是一个构建 Web 应用程序的快速开发框架。 它并没有受到太多大公司的重视，但由于开发速度的原因，创业公司经常使用它。

### <font color=#6495ED size=4 >Java</font>

Java 是 Sun 公司拥有的一种开源语言。可以用来开发桌面或 Web 应用程序，但 Java 使用率在创业公司中被前文说的 PHP 和 Ruby on Rails 抢走很大比例以及在企业（大型公司）中前文讲到的 .NET 占据一定的比例。

### <font color=#6495ED size=4 >ColdFusion</font>

ColdFusion归Macromedia（现为Adobe）所有。它更适合设计人员，但它是一种易于与 Java 集成的脚本语言。 我不推荐学习它，它已经出现了一段时间。

### <font color=#6495ED size=4 >Perl</font>

Perl是一种过时的开源文本解析语言。 目前它仍然在使用，但它更像是一种后端维护脚本的语言。如今，很少有人会用它去构建网站了。

&nbsp;

如果工作中获取收益是你的学习目标，那么在开发人员看来，成为 Web 开发人员是最好的方法。 那里有大量的 ASP.NET，PHP 和 Java 的工作（而且每天都有更多的 Ruby on Rails 的工作）。

Ruby on Rails开发很热门（同时也很有趣）。如果我刚开始，我将用 C＃/ ASP.NET 或 Ruby on Rails 构建一个疯狂的而且很酷的项目，并利用它来完成我的第一个编程工作。在你决定学习哪种语言之前，请先查看所在地区的 Dice，找出哪些语言有着较高的需求量，以及你希望在哪类型的企业工作？

如果你确定你更想成为一名游戏程序员，那就学习微软的 XNA，这是一个用于制作在 Windows  和 Xbox 360 上运行的游戏的入门平台。作为游戏程序员，你最终需要学习 C++，但我并不建议将它作为第一个学习的语言。

## Where Do I Start?  从哪里开始？

你已经知道了编程是什么，并且知道你为何想成为一名编程者。但从未写过一行代码（或者你写过，但不是从高中或者大学开始）。

我是一名自学编程者，但我同样获得了计算机工程师学位，因此我具有双重经验。书籍学习是从实践方面告诉你如何编程。在大学中学习编程是告诉你为何这样做的理论依据。

考虑到这一点，如果我从今天开始学习，我将遵循以下步骤：

1. 购买[ Beginning ASP.NET 3.5: In C# and VB ](http://www.amazon.com/gp/product/047018759X?ie=UTF8&tag=softwarbyrob-20&linkCode=as2&camp=1789&creative=390957&creativeASIN=047018759X) ，从头至尾阅读，实践书本中所有的编程练习。当你完成时，你将能很好的构建一个基础SAP.NET网络应用程序。
2. 挑选一个小项目，自己构建它。范例包含：*一个在线切换的存储器，一个基于网络的联系管理器，或者一个时间跟踪应用程序🆘*。不用担心它们的长相，当下只需要关心它们的功能。编写并且去验证。
3. 一旦完成了功能，则开始关注它的长相设计。
4. 上传到一个主机账户，让它在互联网“活”起来。现在你便拥有了.NET编程经验，同时也为潜在的雇主展示了一个在线的应用程序。

这是万千道路中的一条你可遵循的道路来获取编程经验。我选择.NET是因为现在有巨大的.NET开发者需求。如果你对PHP开发更有兴趣，购买一本PHP相关书籍比如 [ Beginning PHP and MySQL: From Novice to Professional  ](https://www.amazon.com/gp/product/1590598628?ie=UTF8&tag=softwarbyrob-20&linkCode=as2&camp=1789&creative=390957&creativeASIN=1590598628) ，遵循同样的步骤去学习。

对于游戏或者桌面开发者，*坚持上面的会有很大的变化🆘*。方法步骤是不变的，但是同类的书籍和工程却各不相同。

上面的步骤完成了以下内容：

1. *让你编码🆘*。没有什么比阅读学习六个月编码更值得拥有。你应该尽快开始编码。
2. 你将学习一种畅销的语言。真实应用是使用ASP.NET 和 PHP 构建的。学习其中一个语言很有可能在未来几个月让你获得一份工作。
3. 你将建造一个公示块。它可能不是你见过最性感的事，但是它可能是为一个潜在雇主演示的应用程序，同时也将包含简单的代码示范。
4. 完成了步骤2，你将清楚是否喜欢编程。此时据我所知，大多数编程者将不愿意放下书本，废寝忘食的思考它直到完成。如果你发现自己恐惧坐在电脑前花时间来构建找你的工程…很有可能编程并不适合你。当然，*试着区分工作疲惫或者缺乏时间与消极怠工的调调🆘*。

## 已试着做过项目并想成为程序开发人员，下一步如何做？

有很多方式可以来拓展你的编程知识

* 边学边干

  可以找份编程的工作（即便只有 $10 每小时的报酬）。就学习来说，全职开发一个月能抵得上一年的业余开发。
  
* 阅读

  通过阅读来提高编程能力是非常重要的，一个月读 1-2 本书对于新手是必须的（请看后面的书籍推荐），同时也可以针对性地教他们一些高级开发技能

* 在线学习

  网上可以轻易找到各种编程问题的解答，但我更倾向于通过自己从头系统学习来理解一个新的概念。
  
  语言自己网站可以了解一下，如 http://www.php.net/, http://www.asp.net/, http://java.sun.com/ 
  
* 学徒

  成为一个优秀程序员，最好的方式莫过于在资深优秀员的带领下学习，他们不仅能教会你基础知识，还会深入地传授经年积累的经验。但找到一个名师还是相当困难的
  
* 大学

  我的亲身经历告诉我大学教育与实际工作开发有很大的区别。精通时间复杂度评估对实际工作意义不大。大学教育对于提高理论修养很有帮助（有助于你成于高阶程序员），但还是比不上头几年工作实践的作用
  
* 专业培训

  我认识一个出身于培训学校的程序员。虽然对语言足够掌握，有比较强动手能力，但在理论和设计方面颇为不足。
  
  她的代码偏重实用，有大量低效暴力解决问题的方式，写出的代码既缺少设计也不好维护。这是理论知识与解决问题实践的平衡出了问题。
  
  也许只是个例，不应就此否定培训学校。但我想可能优秀程序员并不是培训课教出来的。像网管之类也许才是培训学校的专长
  
## I Just Graduated from School, How Can I Get Experience? 我刚毕业，怎样获取工作经验

获取编程经验最好的方法就是得到一份编程的工作。对于学习来说，没什么比每天工作8小时并且还能赚钱更好的差事了。

但如果你不能立即找到入门级职位的话，你应该在你寻找工作的同时自己做一些项目，或者参与开源项目。

### 自己做项目
编程和其他工程课程的区别是人们是为了兴趣而做。每人为了好玩设计机械系统。但成千上万的人在空闲时间编程只是为了创造新事物的挑战性和完成后的满足感。

自己做东西并不像听起来那么可怕。职员拥有一旦编程知识，你可以在数天之内搭建一个简易网站。每一点经验的积累都会在日后成为职业开发者时起到至关重要的影响。

没有专业的编程经验并不会真的那么难找工作，完全是个小白才会真的难找。

虽然拥有专业的开发经验最好，但你总需要从菜鸟做起。如果在已经在互联网上开发了项目，你找工作的时候将脱颖而出。我不能告诉你我有多被现在的毕业生震撼到，他们表现出来不仅是自学新语言，而且会花时间自己做项目然后上线。

马上就在脑中出现。

### 开源项目
摘自wikiedia(维基百科)https://en.wikipedia.org/wiki/Open-source_software

>开源项目（OSS）开始是作为免费软件的营销活动。OSS可以被定义为那些可读的源代码在软件协议的规定之下可以被其他人获取且满足了开源的定义。这保证了用户去使用，修改，改进软件，并可以重做它，他们经常被开发者在公有领域合作开发。

这意味着任何人，甚至是菜鸟程序员都可以在开源项目上做贡献。你可能听说过Linux,Apache,WordPress,PHP,and Java，这些都是开源项目。

去 http://sourceforge.net/ ， http://www.codeplex.com/ 或者 https://github.com 上转转，刚开始可能会有些难，但这上面有一大堆项目需要你帮忙。选一个看起来有趣的项目然后读上面的资料看看你怎么能参与贡献。你可能需要申请权限或者你直接就可以检查修改了。

不管怎样，把“开源项目贡献者”加入你的简历都能给你加不少分。

## 没上过大学可以做程序员吗？

根据美国劳工统计局数据，约有 [60% 的 web 开发者](http://www.itworld.com/IWD010507cajob) 或者 [68% 的程序员](http://www.bls.gov/oco/ocos110.htm)（包括 web 开发者）有大学以上学历。

那么没有大学学历可以当程序员吗？当然可以！3/10 的程序员以及 4/10 的 web 开发者没有本科学历，即使你没有学历证书，也不影响你喜爱编程。

当我雇佣一个开发者时，关注的内容有这些（重要性从高到低）

1. 专业经验
2. 证书
3. 大学学历
4. 对开源项目的贡献
5. 博客、文章、论文以及演讲

从这方面讲，空有证书而缺少专业技能和经验将是致命的不足。与其花费大量的时间和精力去考取一个你不了解的技术的认证，不如去学习怎样写代码，去开发一个简单的项目，试着去向开源项目作贡献。最后再考虑证书的事情。

业余项目的复杂度和大小可能为你在新手程序员位置上的竞争带来不小的优势，能在一定程度上消弥在学历与证书上的劣势。新手程序员的竞争足够激烈，我也知道很多优势的程序员并非科班出身（可能由其它行业转行作程序员），或只有专科教育，或者仅出于爱好编程。希望这能为你的程序员之路规划有些启发

## What About Offshoring – Will I have a Job in 5 years? 离岸外包-我能在5年内找到一份工作吗？

如果你读过Thomas Friedman的[世界是平的：二十一世纪的简史](http://www.amazon.com/World-Flat-3-0-History-Twenty-first/dp/0312425074/ref=pd_bbs_sr_1?ie=UTF8&s=books&qid=1227040542&sr=8-1)一书，你就会知道离岸外包的炒作--我们所有的编程工作和呼叫中心工作都将外包到海外--并非完全正确。

离岸外包之所以受到了媒体巨大的关注，是因为它创造了一个畅销的故事，并且还有一些数据支持它。然而问题在于媒体的关注太过片面，这些数据从未显示离岸外包为美国创造了多少工作岗位，它们也倾向于逃避我们在美国对于熟练程序员的迫切需求。

如果你跟随戴尔电脑公司进入它们的离岸外包呼叫中心，[你可能会听到](http://www.spiegel.de/international/business/offshore-backlash-luring-customers-with-local-call-centers-a-496396.html)：

    ...由于收到各种客户投诉，内容从外国口音到支持工作的质量，它们甚至将呼叫中心撤回美国。

各公司已经发现，虽然前端的离岸工作工资成本较低，但是这并不是真正的净收益，由于时差、沟通差异、缺少物理会议等其他问题造成的影响阻碍了它们离岸外包的野心。

这并不是说离岸外包在我们的经济中不真实存在，事实上，很少有白领工作能够避免离岸外包。会计，电气工程，机械工程......都在某种程度上被外包。除非你打算成为首席执行官，行政助理（然而同样正在被虚拟助理接管），或者一份你必须亲临现场的工作，否则你的工作岗位都有被离岸外包的风险。

甚至在当前的金融危机下，计算机和数学职业的[失业率](https://www.bls.gov/web/cpseea30.pdf)在2007年也比美国整体失业率低37%，在2008年10月低43%。另外，查看[此图表](http://feit.anu.edu.au/it/IT_Unemployment_Analysis.pdf)，2000年至2004年的数据显示，计算机行业的失业率以稳定的趋势持续大大低于美国的整体失业率。

我们都知道世界正处于金融危机中，但本周我收到了3封来自招聘人员的电子邮件，询问我是否认识可以填补他们岗位空缺的.NET开发人员。另外，我为一个上星期刚收到资金的客户启动了一个新项目。事实上，总是有编程工作要做。

离岸外包是真的吗？对。但是，在我们完善传送技术之前，对于熟练的开发者来说工作总是有的。

## 编程需要先学习数学吗？

编程不一定要非常擅长数学，但编程的许多思想都来源于代数学。

如果你要开发游戏，数学和物理知识还是必不可少的，至少应该有大学数学的水平。而要开发帐务系统，你需要的只是简单的加减类的数学知识。

在我看来如果擅长数学学习并且喜欢解决逻辑问题，则写代码也更顺利，能坚持走更远更长。你可以试试这本书 [how to move Mount Fuji](http://www.amazon.com/gp/redirect.html?ie=UTF8&location=http%3A%2F%2Fwww.amazon.com%2FHow-Would-Move-Mount-Fuji%2Fdp%2F0316778494%3Fie%3DUTF8%26s%3Dbooks%26qid%3D1174413411%26sr%3D8-1&tag=softwarbyrob-20&linkCode=ur2&camp=1789&creative=9325) （微软的智力题集锦），需要培养特别的思考方式。

对数学越了解，那么解决问题时收获的快乐越多。但不要舍本求末，对于一个代码都没写过的人，让他去先学数学无疑是没有意义的。

## How Much Money Can I Make? 我可以赚多少钱？

我从未和程序员一起工作过，除了一个年收入小于40000美元为非盈利组织工作的人。 *That is straight out of college, entry level corporate development.* :sos:

我合作过的大部分初入行的码农都赚5到5.5万美元每年。中级开发人员的收入为6.5万至9万美元，高级开发人员的收入为8.5美元至15万美元（如果他们的能力非常出色的话，则会更多）。

如果你成为一名自由开发人员，那么一名中级自由开发人员，每小时收费50-60美元，高级开发人员每小时收费60-80美元，领域内的专家每小时收费80-150美元（也就是说有受人尊敬的博客，发表文章，或有一些其他方面的专门市场）。

这些是基于我的经验和大型技术中心（旧金山，洛杉矶，西雅图，纽约和波士顿）的企业薪资发展得出的一些大致正确的薪资范围。有关你所在地区的薪资范围，请访问[Salary.com](https://www.salary.com/)并在“薪资向导”中输入“程序员”和你的邮政编码，或在[Dice](http://www.dice.com/)上搜索你的邮政编码并查看开放的职位列表。

## 如何成为优秀的程序员？

就我所知，每个人都有自己的成长路线。

编程与其它大部分工程实践不同。要作为一个电气工程师，你在学校接受教育，毕业后进入公司工作，并在日后某天通过测试拿到证书。没有人会出于兴趣而设计电气系统，但编程却往往出于人的兴趣。因此在学习如何编程这件事上，有更多的方式方法。

下面我列出了一些个人认为特别重要的方面

1. 不停地学习

   你需要有无穷的求知欲。在刚开始时往往每几周就要读过一本编程相关书籍，在半年到一年的全职编程开发后，就要开始读一些概念性的书，比如 [Pragmatic Programmer](http://www.amazon.com/gp/redirect.html?ie=UTF8&location=http%3A%2F%2Fwww.amazon.com%2FPragmatic-Programmer-Journeyman-Master%2Fdp%2F020161622X%3Fie%3DUTF8%26s%3Dbooks%26qid%3D1174412806%26sr%3D1-2&tag=softwarbyrob-20&linkCode=ur2&camp=1789&creative=9325), [Code Complete（代码大全）](http://www.amazon.com/gp/redirect.html?ie=UTF8&location=http%3A%2F%2Fwww.amazon.com%2FCode-Complete-Second-Steve-McConnell%2Fdp%2F0735619670%3Fie%3DUTF8%26s%3Dbooks%26qid%3D1174412704%26sr%3D8-1&tag=softwarbyrob-20&linkCode=ur2&camp=1789&creative=9325), [Facts and Fallacies](http://www.amazon.com/gp/redirect.html?ie=UTF8&location=http%3A%2F%2Fwww.amazon.com%2FFacts-Fallacies-Software-Engineering-Robert%2Fdp%2F0321117425%3Fie%3DUTF8%26s%3Dbooks%26qid%3D1174412762%26sr%3D1-1&tag=softwarbyrob-20&linkCode=ur2&camp=1789&creative=9325)

   我想对于阅读的重要性再强调也不为过，就像在学习初期沉浸于编程一样重要。

2. 注重概念

   一开始需要培养逻辑概念和语言语法，两者一起学习效果更佳。但作为一个优秀的程序员，将很快超出语言语法的范围。例如在我成为专职程序员的前 18 个月内，我用过了诸如 Perl, PHP, Java, ColdFusion, ASP 等各种语言。让我成为一个好的程序员的不是我对每种语言的知识掌握情况，而是我是我对一些概念的理解和重新认识，比如 DRY（Don't Repeat Yourself），破窗效应，代码重用（这都来自于书本 [The Pragmatic Programmer](http://www.amazon.com/Pragmatic-Programmer-Journeyman-Master/dp/020161622X/ref=sr_1_1?ie=UTF8&qid=1227061027&sr=8-1)）

3. 和优秀的程序员交流

   和学习吉它一样，我从那些比我优秀的人那里最终学会了作曲，编程也是一样。

   网络时代你并不需要与本尊面对面，你只需要去阅读大牛的博客（[Scott Guthrie](http://weblogs.asp.net/scottgu/), [Rocky Lhotka](http://www.lhotka.net/weblog/), [Dino Esposito](http://weblogs.asp.net/despos/), [Scott Mitchell](http://www.scottonwriting.net/sowBlog/)），逛编程论坛，阅读他人的代码。

   读代码是件苦差事，但这值得，因为见得越多理解也越容易，将复杂的理论知识直白的展示出来。

4. 一点心得

   我想告诉那些想要成为程序员的人们：世上不缺程序员，缺少的是工程师（engineer）。

   我曾经遇到很多自称程序员的家伙，他们声称可以在 21 天学会 C#（或其它什么语言），认为他们可以在 10 万行代码规模上开发。在我看来，编程只是软件开发的一部分，此外还涉及到

   * 需求分析
   * 系统设计以及设计模式，如 n-tier, MVC 等等
   * 写可测试，可维护的代码
   * 编写测试代码
   * 学习如何重构

   记住，编程只是真正工作的一部分

## FAQ
### 如何成为一个 .NET 程序员

买任意本我之前提到的书，从头读到尾，实现一个动态网站。这是最高效的学习语言的方法，比你花几千块参加培训要有用的多。

### Mac 电脑上如何开始编程实践

Mac 电脑基于 UNIX，可以运行任何兼容 UNIX 的编程语言。本书中提到的 PHP，Ruby，Java 和 Perl 都可以用

个人建议你使用 Ruby 或者 PHP。如果你想做游戏，可以使用 C++（实话说这门语言相当难以上手），可以参考 [Sams Teach Yourself C++ in 21 Days](http://www.amazon.com/gp/product/0672327112?ie=UTF8&tag=softwarbyrob-20&linkCode=as2&camp=1789&creative=390957&creativeASIN=0672327112)

### 我想以程序设计为第二职业，如何快速上手？

如果你已经有工作了，那么应该考虑如何与当下的工作结合。例如，如果你是在大公司做财务相关工作，很有可能你用的软件就是一些程序员在家里写出来的。你可以与那些软件的 CTO 或 CIO 多打交道，请教他们是否能带你入坑。当然你需要提前自学编程，详细见前文。

在日常使用 Excel 或 Access 中，你也可以发挥你的程序设计能力，其中涉及到很多高端的编程和数据库知识

### 哪种编程语言最简单、最易上手？

Visual Basic（现在改名为 VB.NET）简单易学，同时也足够强大。你可以用更早的语言如 Pascal 或 BASIC，但这些语言已经不怎么被使用了，为了实用不建议学习。

VB.NET 在 ASP.NET 中常用，你可以看这本 [Beginning ASP.NET 3.5](http://www.amazon.com/gp/product/047018759X?ie=UTF8&tag=softwarbyrob-20&linkCode=as2&camp=1789&creative=390957&creativeASIN=047018759X)，学习如何使用 VB.NET 和 C# 来构建动态网页

### 想成为系统开发程序员，应该学习什么语言

系统软件与计算机硬件相关，通常以服务形式提供，也就是说不是面向普通用户的。例如磁盘碎片清理或者设置驱动开发等工作。作为系统开发人员，你需要学习 C 或者 C++

### 我想成为优秀程序员，但担心没有天赋，是否要坚持？

这个问题不好回答。我认为你应该试试看你是否真的喜欢编程。如果在起步阶段遇到很大的困难，那也不必勉强。作为程序员，你会经常面对一些负面的情况，如不懂事的老板，无止境地加班，长期坐在电脑前面。没有爱好是很难计坚持下来的。

### 针对点对点程序（Peer to peer）开发有没有好的建议？

不同于通常的 C/S 架构，点对点是去中心服务器的，应用之间可以相互通信，这类应该如 Napster 或其它即时通讯工具。

.NET 和 Java 都可以用于这类开发。.NET（C# 或 VB.NET）可以在 Windows 系统上运行，而 Java 相对上手困难一些，但可以在很多系统上运行（Windows,Linux,Mac 等）


### 什么是敏捷开发（Agile Development）？

你可能经常听到敏捷开发这个词，维基百科是这样定义的

> 敏捷开发是指一类软件开发方式。敏捷开发方式通常提倡：经常性的自省和改良项目管理方式；鼓励团队合作、自我组织和统计的领导方式；一系列快速迭代生成高质量软件的最佳实践方式；将开发与用户需求以及公司前景统一起来的商业动作方式

换种方式来说，敏捷开发要求你不要一开始就设计十分详细的规划，并一成不变地执行计划，而应该以更灵活、自由的方式进行

### 在开始一个项目前是否应该在纸面上写好规划？

在项目开始前，你可以写一个规划设计文档，简洁粗糙也可以。但我不建议你把代码写在纸上，再照着敲进电脑。许多年前也许有人这么做，但那都是过去了。

### 学习编程的最大收获是什么？

对我来说最大的收获是能够在任何国家地区生活，赚足够的钱，为自己工作，从无到有的创建新的事物

### 如何找家软件公司来培养我的能力？

我在之前的文章 [Does Anyone Know of a Real Software Apprenticeship?](http://www.softwarebyrob.com/2008/07/29/does-anyone-know-of-a-real-software-apprenticeship/) 中收到一些建议。有一些实习职位，但很难找到真正能教导你如何编程的实习工作。大多数情况下，你会被安排一个项目，扔给一本语言相关的书，就这样而已。

我还没有听过有人做广告来招收程序员学徒的，多数情况下都是随着项目接受资深程序员的指导。因此我的建议是你在面试时去打听打听公司有没有新人培养文化。

### 计算机用户成为程序员有多难？相比于那些极客

当我成为专业程序员时，我也不能被称为极客。你不需要成为极客才能写代码，而且日常的写代码工作与极客如关，如网页设计、工具开发等

也许我现在可以被称为极客，所以小心我带坏你们 ^_^

### 计算机科学（Computer Science）与编程（Programming）有什么区别？

在我的 [Advice on How to Become a Programmer](http://www.softwarebyrob.com/2007/03/20/advice-on-how-to-become-a-programmer/) 一文中说得很清楚了

> 计算机科学不是编程，就像材料科学不同于建筑一样。不幸的是，许多学生在完成一半学业时也分不清其中区别。如果你只沉溺于纯粹的数学理论或复杂模型建立，那你应该从事计算机科学。如果你喜欢写代码，那就从事编程开发吧

计算机科学主要处理计算机软件理论以及其法。理论上你可以取得一个计算机科学的学历，但没有也不会写一行代码
